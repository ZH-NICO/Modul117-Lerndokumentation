# Theorie-Factsheet - Tag 1

## Thema

**Übersicht:** Einführung Modul 117 – IPERKA-Vorgehensmodell und Netzwerkaufbau/-dienste

## Kernkonzepte

### 1. IPERKA-Vorgehensmodell

**Erklärung:** IPERKA ist eine Methode um Projekte Schritt für Schritt zu bearbeiten. Die 6 Phasen sind: **I**nformieren, **P**lanen, **E**ntscheiden, **R**ealisieren, **K**ontrollieren, **A**uswerten. Jede Phase baut auf der vorherigen auf.

### 2. Lokale Netzwerkdienste

**Erklärung:** Ein LAN (Local Area Network) bietet verschiedene Dienste an: **Filesharing** (Dateien teilen), **Printsharing** (Drucker teilen) und **Internetzugang**. Diese Dienste bestimmen, was das Netzwerk können muss.

### 3. Netzwerkgeräte

**Erklärung:** **Switch** verbindet Geräte im selben Netzwerk. **Router** verbindet verschiedene Netzwerke (z.B. LAN mit Internet). **Accesspoint** ermöglicht WLAN-Verbindungen.

## Wichtige Begriffe

| Begriff | Definition |
|---------|-----------|
| IPERKA | Vorgehensmodell mit 6 Phasen zur strukturierten Projektarbeit |
| LAN | Local Area Network – lokales Netzwerk auf begrenztem Raum |
| Filesharing | Dateien über das Netzwerk gemeinsam nutzen |
| Printsharing | Drucker über das Netzwerk gemeinsam nutzen |
| Switch | Gerät, das mehrere Computer im Netzwerk verbindet |
| Router | Gerät, das Netzwerke miteinander verbindet (z.B. LAN ↔ Internet) |
| Accesspoint | Gerät für drahtlose Netzwerkverbindung (WLAN) |

## Zusammenfassung

- IPERKA hilft, Projekte strukturiert und planvoll abzuarbeiten
- Ein kleines Unternehmen braucht mindestens File-, Print- und Internetdienste
- Switch = im Netzwerk verbinden, Router = Netzwerke verbinden, Accesspoint = WLAN
- Vor dem Bauen muss man zuerst informieren und planen
